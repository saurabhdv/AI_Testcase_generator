6.1. Creating a New CA

This task is used to create a new self-signed or secondary CA.

6.1.1. Prerequisites

The following task requires MSO signatures to be completed. Both officers must have the following roles:

  • Use AWB
  • CA and Key tasks

A connection to the CM host must have been established (refer to Section 3.2, “CM Host Connections”).

The following information is required by the CMO during the task:

  • The CA name that will appear in the Authority Hierarchy in the explorer bar

  • The name of the issuing CA when creating subordinate CAs.

  • CA key to be used for the certificate (refer to Section 7.1, “Creating a New CA Key”).

  • The signature algorithm

  • The distribution rules applicable.

  • The certificate format applicable.

Note

It is recommended that formats, which are not available, be generated before performing this task.

6.1.2. Options

Clicking the Save button at any time during the creation of the Authority, prior to clicking the OK button, will save the data and place the incomplete Authority request in the Authority Hierarchy. Authority creation can be completed at a later stage by highlighting the unsigned Authority in the explorer bar and selecting the Modify option from the Edit menu, the toolbar, or the right-click shortcut menu.

The certificate attributes displayed in the Create Authority Request dialog can be customized. This is detailed in Section 6.1.3.1, “Customizing the Attribute Display” at the end of the procedure.

Note

It is possible to define delayed publishing when a new Authority is created if steps 15 and 16 in the following procedure are bypassed.

6.1.3. Procedure

  1. In the AWB window, select Authority from the New menu. The Create Authority Request dialog appears.

  1. Enter the CA name that should appear in the Authority Hierarchy in the explorer bar. This field is mandatory.

  2. Set the CA State to Active or Closed as required.

  3. Select Domain and Visible in subdomain if applicable.

  4. Select the Authority type CA.

  5. Modify the CA Valid from and Expiration date fields by highlighting in turn the days, hours, and minutes, and adjusting the amounts using the up and down arrows. The individual units of date and time may also be entered manually.

    Note

    If the CA is to be a subordinate CA, issued with a standard format definition, the expiration date will be truncated to the expiration date of the issuing CA if necessary.

  6. If the CA is to be a self-signed CA leave the Issuing CA - Self signed box checked (default) and go to step 10.

  7. If the CA is to be a subordinate CA uncheck the Issuing CA - Self signed box. The Issuing CA field is enabled. This field is now mandatory. Click on the browse button to open the Select Authority window.

  8. Click on the required CA to highlight it and click OK. The selected CA appears in the Issuing CA field.

  9. Select the CA usage by checking the Certificate signing and/or the CRL signing check boxes. Selection of at least one option is mandatory.

  10. Click on the Key browse button to open the Select key window.

  11. Click on the required key to highlight it and click OK. The selected key appears in the Key field. This field is mandatory.

  12. Select a key algorithm, from those available in the Key algorithm drop-down list, if an RSA key was selected in 12.

    Note

    The key algorithm is used to restrict an RSA key to be used only for RSASSA-PSS signatures or only with the selected RSASSA-PSS parameters. The drop-down list contains only those algorithms that matches the specified algorithmOID for the CIS device that holds the selected key, see the device configuration in cis.conf. The selected algorithm will be set in the subjectPublicKeyInfo field in the issued CA certificate.

  13. Select a signature algorithm from those available in the Signature algorithm drop-down list.

    Note

    The signature algorithm drop-down list contains only those algorithms that matches the key algorithm for the key for the selected issuing CA, i.e. self-signed or the selected CA.

Warning

if the hashInCis property is set to true and a signAlgorithm or signMechanism is specified for the device that holds the selected CA key, see the device configuration in cis.conf, the selected signature algorithm must be the same as the algorithm specified for the device in cis.conf. No warning message is displayed if any other signature algorithm is selected.

  1. Click on the + button associated with Distribution rules to open the Select Distribution Rules window if immediate publishing is wanted. To specify delayed publishing skip steps 15 and 16.

  2. Select the required distribution rule for the CA certificate from the list in the window and click OK. Repeat steps 15 and 16 until all the relevant distribution rules have been entered. The processing order of distribution rules can be changed by selecting the procedure and using the and buttons. The distribution rules will be processed in the order selected and then stored in CMDB.

  3. Click on the Format browse button to open the Select Certificate Format window.

  4. Click on the required format to highlight it and click OK. The selected certificate format appears in the Format field. This field is mandatory. See Section 3.4.1.1, “Format Selection” for more information.

  5. Select the required Country attribute from the drop-down list in the certificate attributes section of the Create Authority Request dialog.

  6. Enter a new CA name in Common Name. The CA name entered here is the name that will appear in the CA certificate .

  7. Enter the Organization and Organizational Unit names.

    Note

    For customizing the attribute display, please refer to Section 6.1.3.1, “Customizing the Attribute Display” for more information. Also, if Subject Information Access attribute is chosen, please refer to Section 6.1.3.2, “Subject Information Access” for more information.

  8. Click the Ok button and sign the request as described in Chapter 17, Signing of creation and modification tasks.

6.1.3.1. Customizing the Attribute Display

It is possible to customize the set of attributes that are presented in the Create Authority Request dialog.

  1. Click the browse button to the right of the certificate attribute fields. The Fields Chooser dialog box appears.

  1. Select which attributes to present and click OK. The order of the presented fields is fixed. The attributes will be shown in the same order as they are listed in this dialog box.

Note

The Auto add data fields option is used to present all available fields of an existing certificate. It is not applicable in the Create Authority Request dialog .

6.1.3.2. Subject Information Access

For information concerning the subject information access extension, please refer to RFC 5280 - Internet X.509 Public Key Infrastructure Certificate and CRL Profile.

  1. Click on the + button associated with the Subject Information Access field to open the fields below.

  2. Enter the Access method OID to be used.

  3. Select either the URI or E-mail option button in the Access Location section and enter the required information.

  4. Click Finish button.

  5. Click OK button to close the dialog.
